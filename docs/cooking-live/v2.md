Cooking Live Mode — Design System
Parrillero Pro · Real-time cooking operating system · Mobile-first

Design Principle
One glance. Zero thinking. Full control. Every decision below enforces this.
1. Screen Architecture
Six fixed vertical zones. Each zone has exactly one job. No zone competes with another.

Iron rule
Bottom nav collapses to 2 actions during Live Mode. All other nav items are hidden. Return to full nav only after cooking ends.

Transition rule
Every step transition takes 400ms total: 200ms fade-out current, 200ms fade-in next. The timeline scrubber segment animates to filled in 600ms (slightly longer for satisfaction).

4. Color System — Heat Language
Every color maps to a cooking reality. Users learn it in one session. It becomes muscle memory.

Color Role	Hex Target	When Used	Cooking Meaning
Idle	#71717a (zinc-500)	Pre-start, paused	Cold. Nothing is happening.
Active	#f97316 (orange-500)	Main cooking steps	Heat. Fire. Action.
Urgent	#eab308 (yellow-500)	Final 20% of step	Almost done. Pay attention.
Rest	#818cf8 (indigo-400)	Rest/reposo steps	Cool down. Calm. Patience.
Complete	#10b981 (emerald-500)	Step done, end of cook	Perfect. Done. Eat.
Critical	#ef4444 (red-500)	Overdue / missed step	Danger. Act now.
Background	#030303	Always	Neutral canvas. Never fights content.
Color rule — never mix heat and rest
Orange and violet must never appear simultaneously. When entering rest state, orange fades out over 800ms before violet fades in. The transition is the visual equivalent of removing meat from the grill.
5. Timer Dial — Core Component
200px
Dial diameter
8px
Ring stroke width
1s
Ring update interval
Element	Value / Behavior
Ring fill	SVG circle stroke-dashoffset animated via CSS. Fills clockwise as time passes.
Center number	MM:SS format. Font size 56px. fontWeight 900. Counts DOWN by default.
Ring color	Follows state machine color: orange → yellow (urgent at 20%) → green (done).
Background ring	Faint stroke at 10% opacity — shows total arc even at 100% remaining.
Pulse at urgent	Ring gains a CSS @keyframes pulse at 1Hz when state = urgent. Amplitude: 2px scale.
Completion	Ring fills fully, then immediately collapses with a scale(1.12) → scale(1) spring.
Tap target	Full dial is tappable. Tap shows: elapsed / total / percentage complete overlay.

Copy rules for step text

Max 4 words	Sella lado 1
No punctuation	No periods, no commas
No hedging language	Never 'intenta', 'puedes', 'quizás'
Temperature only when actionable	«Espera 57°C» not «~55-60°C range»

8. Engagement Mechanics — Flow State Engineering
Reduce cognitive load to zero
The UI answers 3 questions at a glance — NO reading required:

Question	Answered by
What do I do NOW?	Current Step card — 26px bold, one line
How long left?	Timer Dial — the largest element on screen
Am I almost done?	Timeline Scrubber — progress position
Create addictive feedback loops
Four loops that keep users engaged:

Loop	Mechanism
Micro	Timer ticking — satisfying countdown every second
Step	Complete step → flash + segment fills → dopamine hit
Zone	Color change per zone — visual variety across cook
Macro	Timeline shows progress — 'I can see the finish line'
Dirty-hands UX principles
Principle	Implementation
No required interaction	Timer runs automatically. Cook can ignore the phone entirely between steps.
One-tap completion	Single large tap on 'Completar paso' — no confirmation dialog ever.
Accidental tap protection	Undo toast appears for 4s after step marked complete. Disappears silently.
Screen always awake	Acquire WakeLock API on Live Mode entry. Release on exit.
Glance in 1.5 seconds	Timer dial is readable at arm's length. Tested at 60cm distance.
Voice fallback	If Web Speech API available, announce: 'Voltea la carne' at step transitions.

The north-star metric
Session length during cooking. Target: user opens app at grill start, closes app when plating. Zero drop-offs. Every design decision is measured against this.
11. Implementation Phases
Phase	Scope	Components	Unlocks
Phase 1 — Static	Layout + zones + step display	PhoneMockup, StepCard, NextStepPreview, TimelineBar	Manual step-through by user
Phase 2 — Timer	Real countdown + auto-state	TimerDial (SVG), state machine hook, WakeLock	Hands-free passive cooking
Phase 3 — Motion	All microinteractions + transitions	CSS animations, spring physics, confetti	Emotional engagement
Phase 4 — Audio	Step announcements via Web Speech API	useSpeech hook, announcement queue	Eyes-free cooking
Phase 5 — Social	Shareable result card	ResultCard component, share sheet	Viral loop + retention
Build order discipline
Ship Phase 1 in the current sprint. Do not add animation before the data layer is solid. Users prefer a correct timer over a beautiful broken one.
Summary — The 6 Decisions That Matter
#	Decision	Rationale
1	Timer Dial is the largest element on screen	Time is the only thing that matters mid-cook
2	Step text is max 4 words, imperative voice	Dirty hands can read 4 words in 0.5s
3	Bottom nav collapses to 2 buttons in Live Mode	Remove all choices that don't apply right now
4	Colors map to cooking states, not to categories	Language users already speak (fire = orange)
5	Step transitions are celebrated with a full-screen flash	Every completion must feel like a win
6	WakeLock keeps screen on throughout the cook	The phone must be a dedicated cooking device
